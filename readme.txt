背景：当用户发生欠费、合同将要到期、账单日临近等情况时，后台服务端会推送APP通知到客户端。
水电表每天凌晨3点
账单合同相关推送凌晨5点

1.服务端定时器任务触发推送
2.服务端通过用户ID进行定向推送

3.客户端根据用户ID可接受定向推送的通知
4.客户端在 登录系统 至 登出系统 过程中可接受定向推送的通知（包括退出APP状态）
5.客户端从第一次打开APP开始 至 删除APP，可接受面向全部用户发送的通知
6.客户端在登出后，理论上不再接受定向推送信息，但允许有短时间的延迟（视信鸽平台数据处理时间而定，对业务没有大妨碍）
7.客户端在切换账号后，不可收到对上一账号的定向推送信息（允许有响应延迟，视信鸽平台数据处理时间而定，只要不是客户端处理问题就可以了）。
